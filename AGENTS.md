# AGENTS.md

本文件定义了 AI 辅助开发时必须遵守的规则。

## 代码注释规范

每次修改代码时，必须在关键位置添加注释，要求：

- **不能太少**：每个类、每个公开方法、每个复杂逻辑块都需要注释
- **不能过多**：不要对简单的 getter/setter、变量声明等显而易见的代码加注释
- **注释语言**：使用中文注释
- **注释位置**：注释写在被说明的代码上方或行尾

### 注释示例

```kotlin
// 正确：复杂逻辑需要注释
fun startTransfer() {
    // 在目标目录下创建与源文件夹同名的新文件夹
    val newFolder = targetDoc.createDirectory(sourceName) ?: return@thread

    // 遍历源文件夹中的所有文件并逐个移动
    for (file in files) {
        // ...
    }
}

// 错误：简单代码不需要注释
val name: String = "hello"  // ❌ 不要这样
```

## README 修改记录

每次修改项目后，必须在 `README.md` 的「修改记录」章节追加一条记录，格式如下：

```markdown
### YYYY-MM-DD 简要说明

- 修改内容1
- 修改内容2
```

要求：

- 日期使用当天日期
- 简要说明一句话概括本次修改的目的
- 列出具体的修改点
- 记录追加在「修改记录」章节的最顶部（最新记录在最前面）
